# 公開記事レビュータスク

Zenn に公開された記事をレビューし、問題があれば修正する。

## 入力

- Zenn URL: {{ZENN_URL}}
- ローカルファイル: {{FILE}}

## 処理

1. `web_fetch` で公開記事を取得
2. 以下の観点でチェック
3. 問題があればローカルファイルを修正
4. 修正した場合は `git add -A && git commit -m "fix: review fixes for <slug>" && git push` を実行

## レビュー観点

### 1. 画像の表示確認

- 画像が正しく表示されているか（リンク切れがないか）
- alt テキストが適切か

### 2. 用語の一貫性

- 技術用語が記事内で統一されているか
- 新しい技術用語に英語併記があるか（初出時）

### 3. 表記ルール

- 数式が LaTeX 記法 `$...$` で正しくレンダリングされているか
- 全角・半角間にスペースがあるか
- カッコが半角になっているか

### 4. 日本語の自然さ

- 直訳調で不自然な表現がないか
- 冗長な表現がないか

### 5. 避けるべき表現

| 避ける表現 | 推奨 |
| --- | --- |
| 悪いイベント | 望ましくないケース |
| ブックキーピング | 整合性管理 |
| 0 中心の | 平均 0 の |

## 出力

- 問題なし: 「レビュー完了: 問題なし」
- 修正あり: 修正内容のサマリを出力
